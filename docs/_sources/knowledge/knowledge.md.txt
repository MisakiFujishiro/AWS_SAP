# ITの基礎知識
## CORS
ウェブを構成する上でセキュリティ上の重要な考えがCORS。

まず、セキュリティ上のリスクについて理解するために、同一オリジンポリシーについて説明する。  
同一オリジンポリシーとは、同じオリジン（ドメイン、プロトコル、ポート）からのリクエストについては許可をするものの、異なるオリジンからのリクエストについては拒否するという考えである。  

なぜ、異なるオリジンからの通信を拒否する必要があるかというと、以下のような攻撃を可能にしてしまう。
```
悪意あるサイトAが、ユーザーの知らない間に価値のあるサイトB（例: 銀行）に対してリクエストを送ることができれば、ーザーが銀行サイトにログインした状態で悪意のあるサイトAを訪れると、サイトAは背後で銀行サイトに対して資金移動のリクエストを送ることができる。
```

同一オリジンポリシーはオリジン間で壁の役割を果たしている。一方で、異なるオリジン間で通信を許可する必要性がある。
事前に通信しあうオリジンがわかっている場合はCORSを利用することで、壁にドアを作成しておいて通信を許可してあげる。

## エラーコード
### 1xx:処理中
### 2xx:成功
### 3xx:リダイレクト
### 4xx:クライアントエラー
400 Bad Request, 401 Unauthorized, 404 Not Found, 429 Too Many Requests
### 5xx:サーバーエラー
500 Internal Server Error, 502 Bad Gateway, 503 Service Unavailable
