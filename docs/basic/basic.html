ソリューションアーキテクトとは¶

ビジネス観点からお客様の実現したいことを議論し、クラウド戦略をリードできる。 加えて、技術的バックグラウンドを持ち、技術者としてもクラウド活用をリードできる。

AWSの公式サイトの定義

お客様訪問先でAWSを語るに相応しい人材であり、お客様のシニアクラスな方々と議論した上で、お客様のクラウド戦略に影響を与え、クラウド戦略・クラウドジャーニーをリードします。 一方で、技術的に確固たるバックグラウンドを持ち、お客様のアーキテクトおよびソフトウェア開発者の方々と技術的な議論を交わし、実現可能性を顧客とともに確認しながら、クラウドを利用した移行やイノベーションをリードするポジション

  • 金額：300USD（バウチャーがおすすめ）

複雑な組織に対応するソリューションの設計¶

マルチアカウントなどに対応しつつ、ネットワークやセキュリティなどの観点を正しく導ける能力

新しいソリューションのための設計¶

AWSを導入する案件の運用上の優秀性やセキュリティ、信頼性、パフォーマンスを正しく導ける能力

既存のソリューションの継続的な改善¶

既に導入されている案件の運用上の優秀性やセキュリティ、信頼性、パフォーマンスを正しく向上させる能力

ワークロードの移行とモダナイゼーションの加速¶

移行を実施するにあたる評価やアプローチ、移行後のアーキテクチャを正しく導ける能力

問題を解くときのコツ¶

  • 1問2分を目安、15問ずつ30分*5セットのペース、最後の30分を復習にすることを目指す

  • 問題文が長いときは最後の問いの部分や選択肢を軽く読んでから全体を確認する。

  • 要件が複数ある場合は、一つ一つの要件に切り分ける

  • 正解になりそうな選択肢が複数あるときに絞るために重視されている要件を見返す

    • 最もシンプル

      • 開発が必要なものはアウト

      • コマンドを実行する場合は失敗する可能性があるのでアウト

      • 過剰な要件をみたいしているものはアウト

    • 素早くコストをかけずに

      • ベストプラクティスがお金がかかる場合は、別の選択肢が正解になる可能性もある

    • 「最適」はいろいろな観点があるが以下を意識する

      • コスト

      • 自動化

  • 選択肢のうちどちらも正しそうなときは問いの本質以外の部分で間違っていないかも確認する

  • 沢山の要件があるときは、それぞれの要件をみたいしていない選択肢をひとつづつ消していく

勉強方法¶

WS認定資格試験テキスト&問題集 AWS認定ソリューションアーキテクト - プロフェッショナル
C02に対応している貴重な本。かなり参考書よりの本で最初に通読して、理解するとよい。分厚いが、説明もわかりやすいのでストレスなく読める。ラストの模擬試験の難易度はかなり本番に近いと感じた。繰り返し解いて復習する価値がある。

AWS認定ソリューションアーキテクト-プロフェッショナル
ほぼほぼ問題集の参考書。C01対応だが、本書籍で学んだ内容も本番の試験で問われた。各章の問題も最後の章末問題も本番と同程度のレベルの問題になっていると感じた。
